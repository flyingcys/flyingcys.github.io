ESP32-Series集成esptool-js完整方案

第一步：添加esptool-js依赖

方案A：CDN方式（推荐）

<script src="https://unpkg.com/esptool-js@latest/bundle.js"></script>

方案B：本地文件方式

# 下载esptool-js到本地 # 1. 从GitHub下载最新版本 # 2. 将bundle.js文件放到web_serial/third_party/目录 # 3. 在HTML中引用本地文件 <script src="./third_party/esptool-js/bundle.js"></script>

第二步：修改index.html

在web_serial/index.html的<head>标签中添加esptool-js引用：

<!-- 在现有的script标签之前添加 --> <script src="https://unpkg.com/esptool-js@latest/bundle.js"></script> <!-- 然后是现有的脚本 --> <script src="script-clean.js"></script> <script src="downloaders/downloader-manager.js"></script>

第三步：验证集成效果

现在您可以测试ESP32-Series功能：
  • 打开web_serial/index.html
  • 选择"固件下载"选项卡
  • 目标设备选择"ESP32-Series (自动检测)"
  • 上传固件文件并点击"开始下载"

ESP32-Series下载器特性

✅ 已实现的功能：

  • 自动芯片检测 - 无需手动选择具体型号
  • 智能波特率 - 自动优化到921600bps提高传输速度
  • 实时进度 - 精确的下载进度显示
  • 芯片信息显示 - MAC地址、芯片特性、Flash大小等
  • 错误处理 - 完善的错误提示和恢复机制

🔧 支持的芯片型号：

  • ESP32 (原版)
  • ESP32-S2
  • ESP32-S3
  • ESP32-C3
  • ESP32-C6
  • ESP32-H2

📋 主要接口：

// 创建下载器实例 const downloader = new ESP32SeriesDownloader(serialPort, debugCallback); // 初始化并检测芯片 await downloader.initialize(); // 获取芯片信息 const chipInfo = downloader.getChipInfo(); // 下载固件 await downloader.downloadBinary(binaryData, address); // 擦除Flash await downloader.eraseFlash(); // 清理资源 await downloader.cleanup();

调试和故障排除

常见问题：

  1. ESPLoader未定义 - 检查esptool-js是否正确加载
  2. 串口连接失败 - 确保设备正确连接且串口未被占用
  3. 芯片检测失败 - 检查设备是否进入下载模式

调试技巧：

// 在浏览器控制台中检查esptool-js是否加载 console.log(typeof ESPLoader); // 应该输出 "function" // 启用详细日志 const downloader = new ESP32SeriesDownloader(port, (info) => { console.log('[ESP32-Series]', info); });
